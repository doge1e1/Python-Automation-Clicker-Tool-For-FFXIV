# 🎮 Python Automation Tool for FFXIV

針對 FFXIV 重複性操作設計的 Python 桌面自動化工具，提供兩種 UI 模式：

| 工具 | 定位 | 適合場景 |
|------|------|----------|
| `main.py` | 固定流程自動點擊器 | 已知固定座標、需要倒數計時與剩餘時間估算的批次作業 |
| `tea.py` | 操作錄製器 | 流程複雜、需要錄製完整鍵盤 + 滑鼠操作並匯出重播 |

---

## 📦 安裝依賴

```bash
pip install pyautogui pynput
```

| 套件 | 用於 | 說明 |
|------|------|------|
| `pyautogui` | main.py | 滑鼠點擊控制 |
| `pynput` | tea.py | 鍵盤 & 滑鼠監聽 / 控制 |
| `tkinter` | 兩者 | GUI 介面（Python 內建） |
| `threading` | 兩者 | 背景執行，避免 UI 凍結 |
| `json` | 兩者 | 座標 / 腳本的儲存與讀取 |

## 🖱️ main.py — 固定流程自動點擊器

### 功能說明

針對「製作 → 等待巨集執行」這類固定流程設計，支援雙按鈕與三按鈕兩種模式，並提供倒數計時與剩餘時間估算。

### 介面

```
┌──────────────────────────────────────────┐
│  自動助手                                 │
│                                          │
│  --- 模式選擇 ---                         │
│  ● 雙按鈕 (製作 -> 巨集1)                 │
│  ○ 三按鈕 (製作 -> 巨集1 -> 巨集2)        │
│                                          │
│  --- 座標設定 ---                         │
│  [ 錄製座標 (請看終端機) ]                 │
│  目前座標：製作:(1549,967) | 巨集1:(1859,968) │
│                                          │
│  --- 執行與延遲參數 ---                    │
│       執行總次數: [ 10  ]                 │
│   製作後等待 (秒): [ 1.0 ]                │
│  巨集 1 等待時間: [ 20.0 ]                │
│  巨集 2 等待時間: [ 20.0 ]                │
│                                          │
│  --- 執行進度 ---                         │
│  [1/10] 動作: 點擊製作                    │
│  製作後等待: 0.8s                         │
│  預計總剩餘時間: 03:21                    │
│                                          │
│  [  開始執行  ]   [  終止腳本  ]          │
└──────────────────────────────────────────┘
```

### 執行方式

```bash
python main.py
```

### 使用流程

1. 選擇模式（雙按鈕或三按鈕）
2. 點擊「錄製座標」，依照終端機提示，將滑鼠移至各按鈕位置後按 Enter
3. 設定執行次數與各階段等待時間
4. 點擊「開始執行」，程式會倒數 3 秒讓你切換至遊戲視窗
5. 執行中可隨時按「終止腳本」停止

### 座標儲存

錄製的座標會自動存入 `coords.json`，下次開啟時自動載入，不需重新錄製。

```json
[[1549, 967], [1859, 968], [1900, 920]]
```

### 參數說明

| 參數 | 說明 |
|------|------|
| 執行總次數 | 整個流程要跑幾輪 |
| 製作後等待 | 點擊製作按鈕後，等待動畫完成的秒數 |
| 巨集 1 等待時間 | 點擊巨集 1 後等待執行完畢的秒數 |
| 巨集 2 等待時間 | 點擊巨集 2 後等待執行完畢的秒數（三按鈕模式專用） |

## 🎥 tea.py — 操作錄製器

### 功能說明

參考 BlueStacks 內建腳本錄製功能設計，錄製一段時間內的完整鍵盤與滑鼠操作（含時間戳記），支援匯出 / 匯入 JSON 腳本檔並重複播放。

### 介面

```
┌──────────────────────────────────┐
│  操作錄製器 (支援匯入匯出)         │
│  狀態: 待機中                     │
│                                  │
│  💡 錄製建議                      │
│  請將焦點目標設定在螢幕左邊        │
│  角色位置請接近螢幕右邊             │
│                                   │
│  🎥 錄製控制                       │
│  [ 🔴 開始錄製 (F9)  ]             │
│  [ ⏹️  停止錄製 (End) ]            │
│                                   │
│  📂 檔案管理                       │
│  [ 💾 匯出錄製檔 (Save JSON) ]     │
│  [ 📁 匯入錄製檔 (Load JSON) ]     │
│                                   │
│  ⚙️ 播放設定                       │
│  播放次數: [  1  ]                 │
│  間隔(秒): [ 1.0 ]                 │
│  [ ▶️  開始播放        ]           │
│  [ 🚫 停止播放 (DEL)  ]            │
└──────────────────────────────────┘
```

### 執行方式

```bash
python tea.py
```

### 使用流程

1. 按下「開始錄製」或 **F9** 開始捕捉操作
2. 在遊戲內執行一次完整流程（鍵盤 + 滑鼠皆會記錄）
3. 按下「停止錄製」或 **End** 結束錄製
4. 可選擇「匯出錄製檔」儲存成 JSON 供日後使用
5. 設定播放次數與間隔後，按「開始播放」重複執行
6. 播放中隨時可按 **Delete** 鍵或點按鈕停止

### 全域熱鍵

| 熱鍵 | 功能 |
|------|------|
| `F9` | 開始錄製 |
| `End` | 停止錄製 |
| `Delete` | 停止播放 |

> 熱鍵為全域監聽，在任何視窗下皆有效，操作遊戲時不需切換回工具視窗。

### 播放設定說明

| 參數 | 說明 |
|------|------|
| 播放次數 | 填入正整數執行指定次數；填 `0` 代表無限循環 |
| 間隔(秒) | 每輪操作結束後的等待時間 |

### JSON 格式說明

```json
[
    {
        "type": "mouse",
        "x": 968,
        "y": 223,
        "button": "Button.left",
        "pressed": true,
        "time": 0.548
    },
    {
        "type": "key",
        "key": "Key.f10",
        "time": 1.135
    },
    {
        "type": "key",
        "key": "`",
        "time": 1.888
    }
]
```

| 欄位 | 說明 |
|------|------|
| `type` | 事件類型：`"mouse"` 或 `"key"` |
| `x`, `y` | 滑鼠點擊的螢幕座標（mouse 事件） |
| `button` | 滑鼠按鍵：`Button.left` / `Button.right` |
| `pressed` | `true` = 按下，`false` = 放開（mouse 事件） |
| `key` | 按鍵字串（`Key.f10`、`` ` ``、`<vk_code>` 等格式） |
| `time` | 距錄製開始的時間（秒），用於還原操作時序 |
